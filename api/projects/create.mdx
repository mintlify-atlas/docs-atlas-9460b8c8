---
title: Create Project
api: POST /v1/admin/projects
description: Create a new project in Weapon-X
---

## Authentication

Requires a token with `projects:write` permission.

## Request Body

<ParamField body="reference" type="string" required>
  Unique identifier for the project (max 100 characters). Used in API calls and SDK initialization.
</ParamField>

<ParamField body="name" type="string" required>
  Human-readable project name (max 255 characters).
</ParamField>

## Response

<ResponseField name="data" type="object">
  The created project object.
  
  <ResponseField name="reference" type="string">
    Project reference identifier.
  </ResponseField>
  
  <ResponseField name="name" type="string">
    Project name.
  </ResponseField>
  
  <ResponseField name="created_at" type="string">
    ISO 8601 timestamp of creation.
  </ResponseField>
  
  <ResponseField name="updated_at" type="string">
    ISO 8601 timestamp of last update.
  </ResponseField>
</ResponseField>

## Example Request

```bash
curl -X POST https://api.weaponx.io/v1/admin/projects \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "reference": "my-app",
    "name": "My Application"
  }'
```

## Example Response

```json
{
  "data": {
    "reference": "my-app",
    "name": "My Application",
    "created_at": "2026-03-05T10:30:00Z",
    "updated_at": "2026-03-05T10:30:00Z"
  }
}
```

## Error Responses

### 400 Bad Request

```json
{
  "error": "Validation Error",
  "details": [
    {
      "message": "\"reference\" is required",
      "path": ["reference"]
    }
  ]
}
```

### 401 Unauthorized

```json
{
  "error": "Unauthorized",
  "message": "Invalid or missing authentication token"
}
```

### 403 Forbidden

```json
{
  "error": "Forbidden",
  "message": "Insufficient permissions: projects:write required"
}
```
