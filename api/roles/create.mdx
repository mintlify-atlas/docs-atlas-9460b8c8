---
title: Create Role
api: POST /v1/admin/roles
description: Create a new role with permissions
---

## Authentication

Requires a token with `roles:write` permission.

## Request Body

<ParamField body="name" type="string" required>
  Unique role name (max 100 characters).
</ParamField>

<ParamField body="permissions" type="array">
  Array of permission strings (max 100 characters each). Defaults to empty array.
</ParamField>

## Response

<ResponseField name="data" type="object">
  The created role object.
  
  <ResponseField name="id" type="string">
    Unique role ID (UUID).
  </ResponseField>
  
  <ResponseField name="name" type="string">
    Role name.
  </ResponseField>
  
  <ResponseField name="permissions" type="array">
    Array of permission strings.
  </ResponseField>
  
  <ResponseField name="created_at" type="string">
    ISO 8601 timestamp of creation.
  </ResponseField>
  
  <ResponseField name="updated_at" type="string">
    ISO 8601 timestamp of last update.
  </ResponseField>
</ResponseField>

## Example Request

```bash
curl -X POST https://api.weaponx.io/v1/admin/roles \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "developer",
    "permissions": [
      "projects:read",
      "configs:read",
      "configs:write",
      "rules:write"
    ]
  }'
```

## Example Response

```json
{
  "data": {
```

## Available Permissions

### Project Permissions

- `projects:read` - View projects
- `projects:write` - Create, update, and delete projects

### Environment Permissions

- `environments:read` - View environments
- `environments:write` - Manage environments

### Configuration Permissions

- `configs:read` - Read configuration values and evaluate flags
- `configs:write` - Create and modify configurations
- `rules:write` - Create and assign targeting rules

### Role Permissions

- `roles:read` - View roles
- `roles:write` - Create, update, and delete roles

### Authentication Permissions

- `authentications:read` - View authentication tokens
- `authentications:write` - Create and manage tokens
