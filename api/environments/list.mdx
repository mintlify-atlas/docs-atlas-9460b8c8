---
title: List Environments
api: GET /v1/admin/environments
description: Retrieve all environments
---

## Authentication

Requires a token with `environments:read` permission.

## Response

<ResponseField name="data" type="array">
  Array of environment objects.
  
  <ResponseField name="id" type="string">
    Environment identifier.
  </ResponseField>
  
  <ResponseField name="label" type="string">
    Environment label.
  </ResponseField>
  
  <ResponseField name="region" type="string">
    Region identifier.
  </ResponseField>
  
  <ResponseField name="api_base_url" type="string">
    Base URL for the environment.
  </ResponseField>
  
  <ResponseField name="api_key" type="string">
    API key for the environment.
  </ResponseField>
  
  <ResponseField name="created_at" type="string">
    ISO 8601 timestamp of creation.
  </ResponseField>
  
  <ResponseField name="updated_at" type="string">
    ISO 8601 timestamp of last update.
  </ResponseField>
</ResponseField>

## Example Request

```bash
curl -X GET https://api.weaponx.io/v1/admin/environments \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Error Responses

### 401 Unauthorized

```json
{
  "error": "Unauthorized",
  "message": "Invalid or missing authentication token"
}
```

### 403 Forbidden

```json
{
  "error": "Forbidden",
  "message": "Insufficient permissions: environments:read required"
}
```
