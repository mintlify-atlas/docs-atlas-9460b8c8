---
title: Get Environment
api: GET /v1/admin/environments/:id
description: Retrieve a specific environment by ID
---

## Authentication

Requires a token with `environments:read` permission.

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the environment.
</ParamField>

## Response

<ResponseField name="data" type="object">
  The environment object.
  
  <ResponseField name="id" type="string">
    Environment identifier.
  </ResponseField>
  
  <ResponseField name="label" type="string">
    Environment label.
  </ResponseField>
  
  <ResponseField name="region" type="string">
    Region identifier.
  </ResponseField>
  
  <ResponseField name="api_base_url" type="string">
    Base URL for the environment.
  </ResponseField>
  
  <ResponseField name="api_key" type="string">
    API key for the environment.
  </ResponseField>
  
  <ResponseField name="created_at" type="string">
    ISO 8601 timestamp of creation.
  </ResponseField>
  
  <ResponseField name="updated_at" type="string">
    ISO 8601 timestamp of last update.
  </ResponseField>
</ResponseField>

## Example Request

```bash
curl -X GET https://api.weaponx.io/v1/admin/environments/production-us \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Example Response

```json
{
  "data": {
    "id": "production-us",
    "label": "Production US East",
    "region": "us-east-1",
    "api_base_url": "https://us-east-api.weaponx.io",
    "api_key": "env_prod_abc123...",
    "created_at": "2026-03-05T10:30:00Z",
    "updated_at": "2026-03-05T10:30:00Z"
  }
}
```

## Error Responses

### 401 Unauthorized

```json
{
  "error": "Unauthorized",
  "message": "Invalid or missing authentication token"
}
```

### 403 Forbidden

```json
{
  "error": "Forbidden",
  "message": "Insufficient permissions: environments:read required"
}
```

### 404 Not Found

```json
{
  "error": "Not Found",
  "message": "Environment not found"
}
```
