---
title: Assign Rule to Configuration
api: POST /v1/admin/configs/:key/rules
description: Create and assign a targeting rule to a configuration
---

## Authentication

Requires a token with `rules:write` permission.

## Path Parameters

<ParamField path="key" type="string" required>
  The configuration key to assign the rule to.
</ParamField>

## Request Body

<ParamField body="project_reference" type="string" required>
  Project reference identifier (max 100 characters).
</ParamField>

<ParamField body="name" type="string" required>
  Rule name for identification (max 255 characters).
</ParamField>

<ParamField body="return_value" type="any" required>
  Value to return when conditions match (max 32KB when serialized).
</ParamField>

<ParamField body="conditions" type="array" required>
  Array of condition objects. At least one condition is required.
  
  <ParamField body="attribute" type="string" required>
    The attribute name to evaluate from the filter context.
  </ParamField>
  
  <ParamField body="operator" type="string" required>
    Comparison operator. One of: `EQUALS`, `NOT_EQUALS`, `IN`, `NOT_IN`, `CONTAINS`, `GREATER_THAN`, `LESS_THAN`, `REGEX`.
  </ParamField>
  
  <ParamField body="value" type="string | string[]" required>
    Value(s) to compare against. Use array for `IN` and `NOT_IN` operators.
  </ParamField>
</ParamField>

<ParamField body="rollout_percentage" type="number">
  Percentage of matching users to include (0-100). Defaults to 100.
</ParamField>

## Response

<ResponseField name="data" type="object">
  The created rule object.
  
  <ResponseField name="id" type="string">
    Unique rule ID (UUID).
  </ResponseField>
  
  <ResponseField name="configuration_id" type="string">
    Associated configuration ID.
  </ResponseField>
  
  <ResponseField name="name" type="string">
    Rule name.
  </ResponseField>
  
  <ResponseField name="conditions" type="array">
    Array of condition objects.
  </ResponseField>
  
  <ResponseField name="return_value" type="any">
    Value returned when conditions match.
  </ResponseField>
  
  <ResponseField name="priority" type="number">
    Rule priority (lower = higher priority).
  </ResponseField>
  
  <ResponseField name="rollout_percentage" type="number">
    Rollout percentage (0-100).
  </ResponseField>
  
  <ResponseField name="created_at" type="string">
    ISO 8601 timestamp of creation.
  </ResponseField>
  
  <ResponseField name="updated_at" type="string">
    ISO 8601 timestamp of last update.
  </ResponseField>
</ResponseField>

## Example Request

```bash
curl -X POST https://api.weaponx.io/v1/admin/configs/enable_new_ui/rules \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "project_reference": "my-app",
    "name": "Beta users",
    "return_value": true,
    "conditions": [
      {
        "attribute": "user_segment",
        "operator": "EQUALS",
        "value": "beta"
      }
    ],
    "rollout_percentage": 50
  }'
```

## Condition Operators

### EQUALS / NOT_EQUALS

Exact string match:

```json
{
  "attribute": "environment",
  "operator": "EQUALS",
  "value": "production"
}
```

### IN / NOT_IN

Match against multiple values:

```json
{
  "attribute": "country",
  "operator": "IN",
  "value": ["US", "CA", "GB"]
}
```

### CONTAINS

Substring match:

```json
{
  "attribute": "email",
  "operator": "CONTAINS",
  "value": "@company.com"
}
```

### GREATER_THAN / LESS_THAN

Numeric comparison:

```json
{
  "attribute": "account_age_days",
  "operator": "GREATER_THAN",
  "value": "30"
}
```

### REGEX

Regular expression match:

```json
{
  "attribute": "version",
  "operator": "REGEX",
  "value": "^2\\..*"
}
```
