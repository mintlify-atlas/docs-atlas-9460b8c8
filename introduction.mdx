## What is Weapon-X?

Weapon-X is a high-performance, centralized control system designed to manage application configurations and feature toggles across distributed environments. Unlike standard feature flag systems that suffer from network latency, Weapon-X utilizes a dual-mode evaluation strategy to deliver instant feature decisions.

## How It Works

Weapon-X provides two evaluation modes to match your performance and security requirements:

### Remote Evaluation

The server-side evaluation mode processes feature flag decisions on your backend, providing:

- **Secure evaluation** - Sensitive logic stays on the server
- **Audit logging** - Track every evaluation decision
- **Real-time updates** - Changes take effect immediately
- **Centralized control** - Manage everything from the dashboard

### Sync Evaluation (Zero-Latency)

The client-side evaluation mode downloads configurations once and evaluates locally:

- **Zero network latency** - No API calls during evaluation
- **Offline capability** - Works without network connectivity
- **Reduced server load** - Evaluation happens on the client
- **Cryptographic integrity** - Configs are signed and verified

## Architecture

Weapon-X is built on a modern, scalable architecture:

- **Frontend Dashboard** - React with Vite, Tailwind CSS, and shadcn/ui components
- **REST API Backend** - Node.js with Express and Prisma ORM
- **Database** - PostgreSQL for reliable data persistence
- **TypeScript SDK** - Fully-typed SDK for admin operations and evaluations
- **Docker Support** - Containerized deployment for easy scaling

## Use Cases

Weapon-X excels in scenarios where feature control and configuration management are critical:

- **Progressive Rollouts** - Release features to a percentage of users to minimize risk
- **A/B Testing** - Test different variations with targeted user segments
- **Kill Switches** - Instantly disable features in production without deploying code
- **Environment Parity** - Maintain consistent configurations across environments
- **Multi-Tenancy** - Manage feature access per customer or organization
- **Configuration Management** - Store and retrieve application settings dynamically
