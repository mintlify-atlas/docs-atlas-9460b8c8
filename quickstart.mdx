This guide will take you from zero to evaluating your first feature flag using Weapon-X. You'll deploy the server with Docker, create a project, configure a feature flag, and evaluate it using the SDK.

## Prerequisites

Before you begin, ensure you have the following installed:

- Node.js v20 or higher
- Docker and Docker Compose
- npm, yarn, or bun package manager

<Steps>

### Install the SDK

Install the Weapon-X SDK in your project:

<CodeGroup>

```bash npm
npm install weapon-x-sdk
```

```bash yarn
yarn add weapon-x-sdk
```

```bash bun
bun add weapon-x-sdk
```

</CodeGroup>

### Start the Weapon-X Server

Clone the repository and start the server using Docker Compose:

```bash
git clone https://github.com/LuchoBazz/weapon-x.git
cd weapon-x/server
cp .env.example .env
docker-compose up -d
```

This will spin up:
- **PostgreSQL database** on port `5432`
- **API server** on port `3001`

<Note>
The server uses PostgreSQL 16 Alpine and runs migrations automatically on startup. The default credentials are `weapon_x` / `weapon_x`.
</Note>

### Apply Database Migrations

Run Prisma migrations to set up the database schema:

```bash
bun run prisma:migrate
```

### Create Your First Project

Use the SDK's `AdminClient` to create a project programmatically:

```typescript
import { AdminClient } from 'weapon-x-sdk';

const admin = new AdminClient({
  baseUrl: 'http://localhost:3001',
  headers: {
    Authorization: 'Bearer YOUR_SECRET_KEY'
  }
});

const project = await admin.createProject({
  reference: 'my-app-prod',
  name: 'My Application (Production)'
});

console.log('Project created:', project);
```

<Info>
You can also create projects via the web dashboard at `http://localhost:5173` or using the REST API directly with curl.
</Info>

### Create a Feature Flag Configuration

Create a boolean feature flag configuration:

```typescript
const config = await admin.createConfig({
  project_reference: 'my-app-prod',
  key: 'enable_dark_mode',
  description: 'Toggles the dark mode feature',
  type: 'BOOLEAN',
  is_active: true,
  default_value: false
});

console.log('Configuration created:', config);
```

<Note>
Weapon-X supports four configuration types: `BOOLEAN`, `STRING`, `JSON`, and `SECRET`. Each can have conditional rules for targeted rollouts.
</Note>

### Add a Conditional Rule

Create a rule to enable dark mode for beta users:

```typescript
const rule = await admin.assignRule('enable_dark_mode', {
  project_reference: 'my-app-prod',
  name: 'Beta users get dark mode',
  return_value: true,
  conditions: [
    {
      attribute: 'user_tier',
      operator: 'EQUALS',
      value: 'beta'
    }
  ],
  rollout_percentage: 100
});

console.log('Rule assigned:', rule);
```

### Evaluate the Feature Flag (Remote Mode)

Use `EvaluationClient` for server-side evaluation with audit logging:

```typescript
import { EvaluationClient } from 'weapon-x-sdk';

const client = new EvaluationClient({
  baseUrl: 'http://localhost:3001',
  headers: {
    Authorization: 'Bearer YOUR_SECRET_KEY'
  }
});

const response = await client.evaluate({
  filters: {
    user_tier: 'beta',
    user_id: '12345'
  },
  keys: ['enable_dark_mode'],
  identifier: 'user-12345'
});

console.log('Dark mode enabled:', response.data.enable_dark_mode.value);
console.log('Reason:', response.data.enable_dark_mode.reason);
```

**Response structure:**

```json
{
  "meta": {
    "server_time": 1739347200
  },
  "data": {
    "enable_dark_mode": {
      "value": true,
      "rule_id": "<rule-uuid>",
      "reason": "MATCH"
    }
  }
}
```

### Evaluate with Zero Latency (Sync Mode)

For ultra-low latency, use `SyncEvaluationClient` for client-side evaluation:

```typescript
import { SyncEvaluationClient } from 'weapon-x-sdk';

const syncClient = SyncEvaluationClient.getInstance({
  baseUrl: 'http://localhost:3001',
  headers: {
    Authorization: 'Bearer YOUR_SECRET_KEY'
  }
});

// Sync configurations once (on app startup or periodically)
await syncClient.sync('my-app-prod');

// Evaluate locally with zero network latency
const results = syncClient.evaluate({
  filters: {
    user_tier: 'beta',
    user_id: '12345'
  },
  keys: ['enable_dark_mode'],
  identifier: 'user-12345'
});

console.log('Dark mode enabled (local):', results.enable_dark_mode.value);
```

<Warning>
Sync mode evaluates flags locally without network calls. It's perfect for high-performance applications but won't include audit logging for individual evaluations.
</Warning>

## Evaluation Reasons

When you evaluate a configuration, you'll receive one of these reasons:

| Reason | Description |
|--------|-------------|
| `MATCH` | A rule's conditions matched the provided filters |
| `FALLBACK` | No rule matched; the configuration's default value was returned |
| `DISABLED` | The configuration exists but `is_active` is false |
| `ROLLOUT_EXCLUDED` | Conditions matched but the identifier was excluded by rollout percentage |

## Supported Operators

Use these operators when defining rule conditions:

| Operator | Description | Example Value |
|----------|-------------|---------------|
| `EQUALS` | Exact match | `"beta"` |
| `NOT_EQUALS` | Inverse match | `"free"` |
| `IN` | Value in list | `["US", "CA"]` |
| `NOT_IN` | Value not in list | `["banned", "suspended"]` |
| `CONTAINS` | Substring match | `"@acme.com"` |
| `GREATER_THAN` | Numeric comparison | `"1000"` |
| `LESS_THAN` | Numeric comparison | `"50"` |
| `REGEX` | Regex pattern match | `"^user_.*$"` |
